How it works

The GUI is a thin wrapper — all logic runs in the C engine.

  1. 1GUI asks C engine to scan the project and produce a context pack.
  2. 2User writes a natural language request and selects a backend.
  3. 3C engine prompts the LLM, parses the response, validates the diff.
  4. 4Patch is applied to a sandbox, built, and tested. Logs are saved.
  5. 5GUI displays the result, patch, and full session audit trail.
